## 1) 서비스 콘셉트

- **Starlogue(스타로그)**: _Star + Prologue_  
  “별이 만들어지는 기록”

- **핵심 메타포**: 하루 단위로 공부 기록이 “별”로 변환되고,  **세션 단위의 판정 게임**으로 경험된다.


---

## 2) 핵심 단위: 하루 / 세션

### A. 하루 단위(일일 기록)

- 하루가 끝나면 그날의 결과가 **우주 지도(별/블랙홀)**로 반영된다.

- “하이라이트 리포트”로 요약 콘텐츠가 생성된다.


### B. 세션(Session) 구성 요소

- 공부 시간

- 다짐(목표/서약) 작성

- 태그(과목/주제)


---

## 3) 게임 시스템: 미래의 나와의 베팅

### A. 베팅 개념

- 사용자가 적은 다짐/목표를 바탕으로 “계약(룰)”을 만들고,

- 세션 중 행동(중단/복귀/연속성)에 따라 **승패/보상**을 판정한다.


### B. 패배 시 페널티(흑역사/벌칙)

- 목표 달성 실패 시 AI가 **‘흑역사’(풍자/블랙코미디 톤)**를 생성

- 분량: **30~180자 짧은 박제**(부담 최소화)

- 예시: “오늘 목표 실패 시, AI가 다짐 기반으로 ‘나약한 인간의 일기’를 대신 작성해 박제”


---

## 4) AI 역할 정의: 코치가 아니라 심판

- AI가 “응원/코멘트”를 하는 코치가 아니라,

- **사용자 목표 기반으로 룰을 구성하고**, 이벤트(Stop/Resume)를 근거로 **판정**한다.

- 결과는 “수치”보다 **스토리(경기 리포트)**로 제시한다.


---

## 5) 자원 시스템: 스태미나 / 집중 게이지

### A. 기본 설계

- 세션마다 **스태미나 100** 제공

- **가장 오래 연속 집중한 시간**이 “집중 게이지”로 표시(연속성 가치 부여)


### B. 회복/소모 규칙

- 공부 시간 대비 **50% 이상 연속 공부** 시 스태미나 **+10 획득**

- **중단 시 즉시 n% 소모**

- 스태미나가 **0이 되면 세션은 계속되지만 실패로 간주**


### C. 약속 어김 판정(강한 룰)

- Stop 입력한 예상 시간보다 **더 오래 비우면 “약속 어김 체크”**

- 약속 어김은 **흑역사 재료**로 활용

- **유예 시간 없음**(컨셉상 냉정한 심판)


---

## 6) Stop(중단) 시스템: 사유 + 예상 시간 입력

### A. UX 플로우

- [Stop 버튼 클릭]  
  ├── 사유 선택: **화장실 / 휴식 / 딴짓 / 방해**  
  └── 예상 시간 입력: 예) **5분**


### B. 집중 게이지(연속성) 처리

- 공부 중단 시 **집중 게이지는 스탑**(연속성의 가치 부여)


### C. 사유별 가중치(페널티 차등)

- **화장실 / 방해**: 페널티 낮음

- **휴식**: 중간

- **딴짓**: 높음(자백이므로 리스크 증가)


---

## 7) 화면/연출(UI·캐릭터·이펙트)

### A. 캐릭터 연출(도메인 분리)

- 캐릭터를 별도 도메인으로 만들지 않고,

- **상태 데이터(공부 시간/중단/스태미나)**에 따라  
  **3~5장 내외 애니메이션 이미지**를 프론트 라이브러리로 스위칭


### B. 즉시 보상 UI

- 연속 집중 시간 달성 시 **UI 효과(이펙트)** 제공


---

## 8) 시각화: Git 잔디를 넘어서 “공부 은하수”

### A. 별 규칙

- 하루 공부량 → 별의 **크기/밝기**

- 태그 → 별의 **색상**


### B. Streak(연속 성공) 시각화

- 연속 성공이 이어지면 별과 별이 **선으로 연결**

- 결과적으로 나만의 **공부 은하수(별자리)**가 형성됨


### C. 실패 기록(블랙홀/운석)

- 베팅에서 패배한 날은 별 대신  
  **블랙홀(공간 왜곡) 또는 차갑게 식은 운석** 배치

- 블랙홀 클릭 시 **흑역사 팝업** 노출


### D. 초신성(하이라이트 구간)

- 하루 중 가장 집중력이 높았던 구간은  
  별 주변에 **Nebula(가스 구름) 효과**로 “영광” 표시


---

## 9) 하루 종료 리포트: 경기 하이라이트 편집

### A. 출력 형태(스토리형)

- “오늘의 MVP 구간: 14:10~14:32 (22분 무중단)”

- “위기 순간: 15:05 Stop → 15:12 Resume (유혹 이벤트 발생)”

- “전략 제안: 내일은 시작 10분을 ‘보스 타임’으로 고정”


### B. 기대 효과

- 사용자는 숫자보다 **스토리로 기억**

- 콘텐츠 소비 → **재방문 유도**에 유리
